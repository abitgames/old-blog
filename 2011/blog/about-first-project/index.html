Something About Our First Project

Written on February 22, 2011 by nantas in Blog

We should write about it long long time ago, since we’ve been working on this project long before this blog even exist. Guess that’s because none of us 3 is talky person and used to host a blog.

We were kinda like: “Let’s wait till we finally nail the name of the game”, “Let’s only talk about it till we can present something cool”… Now we decide to put together a version for alpha test and gather some feedback. I would like to pretend to travel back time and tell you what you should already know about it.

When Karas and I decided to start collaborating, I pitched 3 concept to him. One of them is this 3D top-down shooter for iOS. Since we all came from console shooting game industry, we all believe that we can take off the shooter project with ease and can focus more on making and refining a remote collaborating workflow, also on learning Unity.

It was called “Urban Survivors” back then. Because I want the story to happen in a city under zombie attack. As you can see from the theme of our webpage, the story also features a pair of brother and sister, fighting shoulder to shoulder their way through zombies.

We started from July, 2010. Two months later Summercards joined us as our 2D artist. I was a pretty fun learning and experimenting process. At the beginning we were not even sure if we can remote collaborate with Unity indie, but gradually we settle the whole workflow (which I will introduce in future blog post in detail).

Back to the project itself, I wish I could come up with some nice and clean design document in the first place and we stick to it the whole time. But in reality we wanted some innovation on controlling of the game for iPad. Actually the numerous possibility for controlling on iPad is just the point why we want to land on this platform first. Then it turned out the original control scheme is not nearly as smooth as we expected. We have to change the design several times and we’re still not quite sure about the core gameplay part of the game until now.

As it is considered as a “training project” at the beginning, we kept adding new features on the fly, which is really not a good idea. Also we were too excited about the game and started to write the whole story before the core mechanics really comes together. I still love the story very much and will make it happen if we got the resource. But currently we are focusing on a gameplay scene without any narrative and story to nail the core gameplay first.

Later the name of the game and more details will be announced, but if we gonna push it to release or just make it free and move on to another project is still not clear. I guess it will depend on the feedback we get from the alpha version. I will write the postmortem of the first project including more details next time.

See you soon!
