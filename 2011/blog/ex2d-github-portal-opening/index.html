ex2D 1.1.2 Released! Github Portal Opening!

Written on September 11, 2011 by nantas in Blog

Since the release of ex2D 1.1.1, we started using assemble file (dll) for distribution. This will help us dealing with updates in the future much easier. We can release new features and adjust existing code freely without trouble of file structure changes.

(If you haven’t upgrade your ex2D from versions lower than 1.1.0. Please read this webpage for the very important information on upgrading ex2D)

Although assemble file can greatly improve the user experience, we did receive some complains about user not able to review the source code to better understanding the concept and usage of ex2D. To make sure our user get the best out of the product, with the release of ex2D 1.1.2, we opened a Github portal for users to review ex2D runtime classes!

Here’s the link for the repository:Â https://github.com/jwu/ex2D_Runtime

You can examine and learn the code to help you use ex2D to its full extend. Also please notice that you can now report bugs or request new features with this public repository of ex2D! Just locate the “issue” button on the navigation bar. This will lead you to the issues page.

The Github interface is very straight forward and intuitive. I don’t think it’s necessary for me to explain more about how to report a bug or request a new feature. Just click “new issue” in this page and write up what’s wrong or what do you want to see in ex2D. You can apply labels to help us categorize the issue easier but it’s perfectly fine without labels.

We will look into the issues reported and decide if and when we implement them into the build. If we can’t fix an issue or add a feature (or we don’t want to due to design reason) we will label it as “will not do” and close the issue.

Of course we will continue our custom support in the Unity forum thread here:

http://forum.unity3d.com/threads/101811-ex2D-the-best-2D-sprite-solution-for-Unity-RELEASED/

But we will use the Github portal as the official development backlog. So if you posted an issue in the forum thread that we agreed it should be fixed, we will add an issue in the Github portal to keep it on track. You can also browse through the Github portal to see if your request is already in the list before you post a new one. We appreciate it much if you can use this workflow to save our time, so we can add more awesome features to ex2D!
