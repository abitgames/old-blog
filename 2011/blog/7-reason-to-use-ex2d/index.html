7 Reasons ex2D Excels In 2D Game Development For Unity

Written on September 7, 2011 by nantas in Blog

Unity is a 3D engine, but since long time ago people are trying to make 2D games with it. There have been some well established tool out there in the business. Why are we making ex2D when there are other tools available? And why we think ex2D excels as a 2D sprite solution for Unity?

The content of this article is not new, we have added it in the ex2D website for a while. But I’d like to post it here because there’re some of our thoughts about how to make robust and flexible Unity plugin that can fit in most people’s project. Also I’d like to post it in the dev blog so if you think other tools are better or generally disagree with our point, you can leave a comment to tell us

So, what’s the purpose behind the development of ex2D?

We are indie game developer. We want to make excellent 2D games with the most efficient workflow. ex2D is developed with the purpose to maximize user experience and to provide a different approach to Unity 2D game development. Â Here we list some points that we are really happy about for ex2D:

1. ex2D stores 2D sprite/animation in Unity standard assets.

ex2D follows the Unity standards. It treats all user created assets as ScriptableObject, not data in Component. This way, user will create assets such as Texture Atlas, Sprite Animation, Sprite Font files in the project view, and reference them in proper component. ex2D Components are functions/methods to use data, not the data themselves. So you can see we use Sprite component reference Atlas asset for rendering, SpriteAnimation component reference SpriteAnimClip asset for animating.

Some other tools handles asset and functions together in their Component class that can cause problems in a lot of situations. You may want to create a Prefab with sprite, sprite animation and your own gameplay component. What if you want to have multiple instance of the prefab in your scene, with different sprite texture for each one? In other 2D solution Â you have to create multiple prefabs, because the data and functions are non-separable. In ex2D you can use the same prefab all over your game, and assign the proper assets to the component in editing your scene and during runtime!

Also where to store all the ex2D user assets is totally customizable. User can design the folder structure to best fit their needs. Also all the assets can be drag around in the project folder and can be renamed anytime later. And we have far less problem syncing project among several machines this way.

2. ex2D has powerful but user friendly asset editors.

We build custom asset editors to make the creation of assets a pleasure. You can create, adjust this asset all intuitively in Editor Window. Our Atlas Editor have features similar to TexturePacker and Zwoptex, providing more flexible atlas editing ability. The Sprite Animation Editor is the most powerful one in the business and will keep getting better. Just watch our video tutorial and you will see what I mean. And the training time for ex2D editors are less than 10 minutes, that means you can easily toss this tool to your artist and he will come up with something cool in no time.

3. Our code are heap allocation safe.

When we use some other 2D solution, we found there’re runtime heap allocation problems when changing texture or sprite text. If you have a lot of dynamic text, this will make Unity Garbage Collection go nuts and cause a lot of cpu-time on mobile device. ex2D solved the problem by carefully writing and test code on iOS device. We tested every single line of our code including Unity built-in API for heap allocation and will keep doing this for any feature and change in the future.

4. Our exSpriteFont supports Unicode character, and have a technique named AtlasFont that can combine the texture for multiple bitmap font and sprite textures into one Atlas for better performance!

5. Next up we have texture tracking technique.

ex2D store links to textures in a table. When atlas or Sprite Animation changes, ex2D will check the table and update those texture reference automatically. It also checks if Sprite’s reference texture is in Atlas or not, and try its best to use the proper result.

6. Small but very handy components and tools come along like Layer2D component and Group Import Editor.

The Layer2D component give you easier control over the z-index of all your sprites. You can combine character sprites and their shadows together in one prefab and not worry about messed up their z-coordinate when you drag your prefab into the scene. And not to worry about setting the wrong z value when editing a Unity animation. Group Import Editor allows you to batch changing texture and other assets import setting, making importing large amount of image files for your 2D game with ease. (you need to change the texture type to “gui” for each file without the help of ex2D).

More useful small tools are coming such as SoftClip and Scale9Grids. We didn’t invent the original code, but we are happy to implement more small handy features that can make developer’s life easier!

7. Last but not least, our tool set is design and created to insert minimal stuff into your project.

A big and common problem about middleware is that sometimes they try to do too much and includes a lot of classes that you may never need. ex2D only does things you can’t get around in Unity without middleware. We keep our features as essential as possible and build the workflow to closely collaborate with Unity’s built-in tools and classes. So it’s easy to build your own tool for different project needs or integrate other middlewares for the feature you need.

 

That’s for today. We are developing “add-ons” for ex2D to extend the functions to handle tile maps, skeleton animation and what not. It will be in separate packages so users don’t need to pay anything they don’t need. That being said, the reasons for choosing ex2D will be increasing constantly! We are looking forward to it, I hope you are as well

If you are interested in buying ex2D or just want to try it out first, feel free to click the link on the right column. It will take you to the frontpage of ex2D, and you will find more information, download link for the evaluation version and documentation. Have fun!
